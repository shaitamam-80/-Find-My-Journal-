# דוגמאות PRD

## דוגמה מלאה: Find My Journal

```markdown
# Find My Journal - PRD

## סיכום מנהלים
Find My Journal הוא כלי שעוזר לחוקרים אקדמיים למצוא את הז'ורנל 
המתאים ביותר לפרסום המאמר שלהם. הכלי משתמש ב-OpenAlex API 
לניתוח התאמה בין תקציר המאמר לז'ורנלים אקדמיים.

## הבעיה
- **מה הבעיה:** חוקרים מבזבזים שבועות בחיפוש ז'ורנל מתאים
- **למי:** חוקרים אקדמיים (דוקטורנטים, פוסט-דוק, סגל)
- **הכאב העיקרי:** אין כלי פשוט שמתאים בין מאמר לז'ורנל
- **ההשפעה:** עיכוב בפרסום, reject-ים מיותרים, בחירה בז'ורנלים predatory

## הפתרון המוצע
אפליקציית web שמקבלת כותרת ותקציר של מאמר ומחזירה רשימה 
מדורגת של ז'ורנלים מתאימים עם הסבר להתאמה.

## יעדים ומדדי הצלחה
| יעד | מדד | ערך מטרה |
|-----|-----|----------|
| חיסכון בזמן | זמן למציאת ז'ורנל | מ-שבוע ל-10 דקות |
| דיוק | שביעות רצון מההמלצות | 80%+ |
| צמיחה | משתמשים פעילים חודשיים | 1,000 ב-6 חודשים |

## קהל יעד

### פרסונה ראשית: דוקטורנט/פוסט-דוק
- מפרסם 2-3 מאמרים בשנה
- לא מכיר את כל הז'ורנלים בתחום
- רגיש לעלויות (מעדיף OA או מענקים)

### פרסונה משנית: פרופסור ותיק
- מפרסם 5-10 מאמרים בשנה
- מכיר ז'ורנלים אבל רוצה לגלות חדשים
- חשוב לו IF ומוניטין

## User Stories

### Must Have (MVP)
- כחוקר, אני רוצה להזין כותרת ותקציר, כדי לקבל המלצות לז'ורנלים
- כחוקר, אני רוצה לראות הסבר להתאמה, כדי להבין למה הז'ורנל מתאים
- כחוקר, אני רוצה לסנן לפי תחום, כדי לצמצם תוצאות

### Should Have
- כחוקר, אני רוצה לשמור חיפושים, כדי לחזור אליהם
- כחוקר, אני רוצה לייצא ל-CSV, כדי לשתף עם מנחה
- כחוקר, אני רוצה לראות פרטי ז'ורנל, כדי להחליט

### Nice to Have
- כחוקר, אני רוצה התראות על ז'ורנלים חדשים
- כחוקר, אני רוצה להשוות ז'ורנלים
- כחוקר, אני רוצה dark mode

## מה לא בונים (Out of Scope)
- **Submission Management** - לא מנהלים הגשות, רק ממליצים
- **Impact Factor חי** - נתונים לא זמינים ב-API חינמי
- **Acceptance Rate** - נתונים לא זמינים
- **אפליקציית מובייל** - Web בלבד ב-MVP

## זרימת משתמש (User Flow)
1. משתמש נכנס לדף הבית
2. מזין כותרת ותקציר (או מעלה PDF)
3. לוחץ "Find Journals"
4. מקבל רשימה מדורגת של 20 ז'ורנלים
5. לוחץ על ז'ורנל לפרטים נוספים
6. (אופציונלי) שומר/מייצא תוצאות

## דרישות טכניות
- **פלטפורמה:** Web (React + FastAPI)
- **אינטגרציות:** OpenAlex API
- **אחסון:** Supabase (PostgreSQL)
- **Authentication:** Supabase Auth

## סיכונים ואתגרים
| סיכון | הסתברות | השפעה | מענה |
|-------|----------|-------|------|
| OpenAlex API מוגבל | בינוני | גבוה | Caching אגרסיבי |
| דיוק נמוך | בינוני | גבוה | שיפור Algorithm + feedback |
| חסמי גישה | נמוך | בינוני | Rate limiting |

## שלבי פיתוח
1. **MVP** - חיפוש בסיסי עם 20 תוצאות - 2 שבועות
2. **V1.0** - הרשמה, שמירת חיפושים, ייצוא - 2 שבועות
3. **V2.0** - Premium features, AI explanations - 4 שבועות
```

## PRD Checklist

לפני שמסיימים PRD, וודא:

### הבעיה
- [ ] הבעיה מוגדרת בבירור
- [ ] יש נתונים/ראיות לבעיה
- [ ] קהל היעד מזוהה
- [ ] הכאב מובן ומתועד

### הפתרון
- [ ] הפתרון עונה על הבעיה
- [ ] ה-Scope ברור (IN/OUT)
- [ ] MVP מוגדר
- [ ] User Stories כתובים

### ביצוע
- [ ] מדדי הצלחה מוגדרים
- [ ] סיכונים מזוהים
- [ ] Timeline ריאלי
- [ ] אין שאלות פתוחות קריטיות
